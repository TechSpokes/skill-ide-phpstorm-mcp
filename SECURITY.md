# Security

## Supported Versions

The `main` branch receives security fixes.

## Reporting A Security Issue

Do not open a public issue for credential leaks, private project exposure, release artifact contamination, or workflow security concerns.

Use the repository owner's private security reporting channel when available.

The author and maintainer is TechSpokes: [www.techspokes.com](https://www.techspokes.com).

## Sensitive Material

Do not commit secrets, credentials, private customer material, proprietary research, local database credentials, or local machine paths as reusable skill guidance.

Raw research, `.intake` material, and disposable fixtures are not release artifacts.
